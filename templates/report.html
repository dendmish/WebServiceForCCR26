1. Business Problem

Customer churn prediction is a critical business task. The goal is to identify customers likely to leave the company and support retention strategies.

2. Dataset

The dataset contains demographic and behavioral customer information.

  • Age
  • Tenure
  • Usage Frequency
  • Support Calls
  • Payment Delay
  • Total Spend
  • Last Interaction
  • Gender

Target variable: Customer Churn

3. Exploratory Data Analysis

Exploratory analysis revealed that customers with higher payment delays and more support calls are more likely to churn.

Figure 1. Distribution of customer tenure. Customers are distributed relatively evenly across different service durations.

Figure 2. Age distribution grouped by churn status. Age alone shows only moderate separation between churned and retained customers.

Figure 3. Tenure versus churn. Customers with longer tenure tend to remain with the company more frequently.

Figure 4. Usage Frequency versus churn. More active customers demonstrate a lower probability of churn.

Figure 5. Support Calls versus churn. Frequent support interactions are associated with higher churn rates.

Figure 6. Payment Delay versus churn. Payment delay demonstrates the strongest separation between churned and retained customers.

Figure 7. Total Spend versus churn. Spending behavior differs between churned and active customers.

Figure 8. Last Interaction versus churn. Customer activity recency influences churn probability.

4. Statistical Analysis

To investigate the statistical properties...

Kolmogorov-Smirnov Test

The Kolmogorov–Smirnov test was used to evaluate whether selected continuous variables follow a uniform distribution.

  • Age
  • Last Interaction

For both variables, the obtained p-values were below 0.05. Therefore, the null hypothesis was rejected, indicating that the distributions significantly differ from a uniform distribution.

Chi-Square Test

Chi-Square tests were performed for several categorical and discretized behavioral features.

  • Usage Frequency
  • Support Calls
  • Payment Delay

All tests produced statistically significant results (p-value < 0.05), confirming the presence of meaningful structure within the data and supporting the usefulness of these variables for churn prediction.

5. Feature Selection

Figure 9. Random Forest feature importance. Payment Delay, Support Calls, Tenure and Usage Frequency were identified as the most influential predictors of customer churn.

Feature importance analysis showed that Payment Delay contributes the most to churn prediction, followed by Support Calls, Tenure and Usage Frequency.


Based on the results of feature selection, the following variables were chosen for further modeling:

  • Payment Delay
  • Support Calls
  • Tenure
  • Usage Frequency
  • Gender_Male

6. Machine Learning Models

Multiple machine learning algorithms were evaluated, including Logistic Regression, Polynomial Logistic Regression, KNN, Random Forest, SVM and LightGBM.

Hyperparameter optimization was performed using GridSearchCV where applicable.

Figure 10. ROC-AUC scores obtained during SVM GridSearch. The RBF kernel consistently outperformed the linear kernel, while increasing parameter C improved model performance.

The best SVM configuration was obtained with kernel = RBF and C = 10, achieving ROC-AUC = 0.9722.

Figure 11. Effect of training sample size on SVM performance. Model quality stabilizes after a relatively small fraction of the training data is used.

Additional experiments demonstrated that SVM performance reaches a plateau after approximately 5–10% of the available training observations, indicating that the dataset already contains sufficient information for effective classification.

7. Error Analysis

A detailed error analysis was conducted to investigate observations misclassified by Polynomial Logistic Regression but correctly classified by KNN.

The objective was to identify nonlinear patterns that could not be captured by linear decision boundaries.

Figure 12. Distribution of Tenure for observations misclassified by Logistic Regression but correctly classified by KNN.

Figure 13. Usage Frequency distribution for observations misclassified by Logistic Regression but correctly classified by KNN.

Figure 14. Support Calls distribution for observations misclassified by Logistic Regression but correctly classified by KNN.

The largest differences between correctly and incorrectly classified observations were observed for:

  • Tenure
  • Usage Frequency
  • Support Calls

These findings indicate the presence of local nonlinear relationships in the data. Such patterns are effectively captured by KNN, while linear models struggle to represent them.

8. Final Results

LightGBM achieved the highest ROC-AUC score.

Figure 15. Performance comparison of all evaluated models.

The comparison demonstrates that nonlinear machine learning algorithms substantially outperform linear approaches on the customer churn prediction task.

LightGBM achieved the best overall performance, followed by Random Forest and KNN.

Best Model: LightGBM

ROC-AUC = 0.9846

9. Conclusion

This project covered the complete machine learning pipeline for customer churn prediction, including:

  • Exploratory Data Analysis (EDA)
  • Statistical testing
  • Feature selection
  • Machine learning model development
  • Performance evaluation and comparison

The most influential factors affecting churn were:

  • Payment Delay
  • Support Calls
  • Tenure
  • Usage Frequency

LightGBM achieved the best overall performance with ROC-AUC = 0.9846 and was selected as the final model.

The trained model was saved in PKL format and integrated into a Flask web application, enabling real-time customer churn prediction through a user-friendly browser interface.
